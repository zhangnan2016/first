# 🇨🇳 ChinaReady（暂定名）· 产品整体大纲

> **文档版本**：v0.1 MVP 大纲
> **生成日期**：2026-06-18
> **关联文档**：[inbound-tourism-market-research.md](./inbound-tourism-market-research.md)
> **技术栈**：Next.js + Spring Boot + MySQL + Spring AI
> **开发周期**：3 周 / 1 人 + AI 协作
> **本期排除**：不做支付交易、不做酒店预订、不做机票比价

---

## 总体判断（MVP 取舍）

3 周 / 1 人 + AI 的预算，做不完「社区 + 攻略 + AI」三角。策略是——**把三角拆成「骨架 P0 + 血肉 P1 + 飞轮 P2」**：P0 必须能独立跑通一个完整价值闭环（来华游客真实用得上、愿意分享），社区作为长期飞轮降级到 P2，用 PGC 种子内容托底。

---

## 一、产品定位

> **一句话定义**：面向英语母语国家自由行游客的「来华旅行即时帮助平台」——出发前用**结构化攻略**做好准备，落地后用 **AI 助手**即时解难，用**英文本地社区**发现真实中国。

| 维度 | 说明 |
|---|---|
| **给谁用** | 美/英/澳/加/欧英语母语国家、中青年、独立自由行游客（初访 + 回头客） |
| **解决什么问题** | 信息获取困难、无英文专属社区、无即时帮助渠道（调研痛点 TOP3） |
| **核心价值** | 让外国游客「**敢来、会用、玩好**」——把"入境连环入门税"和"看不懂"压到最低 |
| **差异化护城河** | 现有玩家无人同时覆盖「结构化攻略 + 活跃社区 + AI 实时助手」三角（调研维度三结论） |

## 二、目标用户画像

**主用户**：英语母语国家、25–45 岁、习惯移动端决策、重视独立探索的自由行游客。

**典型场景（3 个）**：

| # | 场景类型 | 用户画像 | 核心诉求 | 对应模块 |
|---|---|---|---|---|
| A | **入境前焦虑型**（首次背包客） | 第一次来华，出发前 2 周开始研究 | "我落地第一天能不能活下来"——支付、VPN、手机号、地图 | 入境生存清单 + 结构化攻略库 |
| B | **在地迷路型**（入境中求助） | 站在街头看不懂菜单、不会用地图、不知怎么付款 | "此刻、此地、立刻给我答案" | AI 实时助手 + 本地生活榜 |
| C | **深度种草型**（回头客） | 已来过 1 次，想去非游客区、找地道小众店 | "本地人去哪儿、真实评价" | 英文本地生活榜 + 旅行社区 |

## 三、MVP 功能模块（6 个核心模块）

| 模块 | 一句话价值主张 | 核心功能点 |
|---|---|---|
| **① 入境生存清单**<br>Arrival Survival Kit | 解决"入境第一天的连环入门税"——把支付/VPN/手机号/地图做成一份照着做就行的清单 | 1. 城市×阶段化 checklist（出发前/落地/首日）<br>2. 关键操作图文指引（绑卡、装 VPN、装地图）<br>3. 免签政策实时同步（240h/55 国等）<br>4. 离线缓存 + 进度勾选 |
| **② AI 实时助手**<br>AI Travel Assistant | 解决"无即时帮助渠道"——拍照即翻译、提问即解答，覆盖最高价值场景 | 1. 拍照翻译（菜单/路牌/票据）<br>2. 自然语言即时问答（"这家店能用 Visa 吗""怎么去地铁站"）<br>3. 位置 + 行程上下文记忆<br>4. 答案一键复制/分享/保存<br>5. 兜底转真人求助指引 |
| **③ 英文本地生活榜**<br>Curated Local Picks | 解决"无英文版大众点评"——结构化、可评价、英文的找店入口 | 1. 精选榜单（可外卡支付的餐厅/景点/购物）<br>2. 商户卡片（支付方式/营业时间/英文菜单标识）<br>3. 用户评分 + 英文简评<br>4. 城市/品类筛选<br>5. 轻量地图锚点 |
| **④ 结构化攻略库**<br>Structured Guides | 解决"信息碎片化"——把 Reddit 的散乱问答整理成可检索的英文攻略 | 1. 城市/主题攻略（地铁、购票、禁忌、行程模板）<br>2. 结构化目录 + 快速跳转锚点<br>3. 更新时间 + 信息置信度标注<br>4. 一键转 AI 助手追问 |
| **⑤ 旅行社区/问答**<br>Travel Community | 解决"缺乏专属社区"——英文本地化的中国旅行 UGC 与即时问答（长期数据飞轮） | 1. 英文发帖/提问/回答<br>2. 标签体系（城市/痛点/行程阶段）<br>3. 点赞/收藏/排序<br>4. 热门问题聚合<br>5. **PGC 种子内容托底冷启动** |
| **⑥ 账号与个人空间**<br>Account & My Trip | 承载用户身份与留存——个人进度、收藏、分享的基础设施 | 1. 邮箱/Google 登录（**不依赖中国手机号**）<br>2. 我的清单/收藏/提问聚合<br>3. 行程草稿（轻量记录）<br>4. 分享卡片生成（反哺 Reddit/社媒）<br>5. 订阅 + 语言偏好设置 |

## 四、模块优先级

| 优先级 | 模块 | 判定理由 |
|---|---|---|
| **P0 必须有**（3 周必交付） | ① 入境生存清单 | 痛点最深 + 核心获客钩子（SEO 友好） |
| **P0** | ② AI 实时助手 | 差异化最强、价值最显性、是调研点名的 North Star 承载体 |
| **P0** | ⑥ 账号与个人空间 | 基础设施，承载留存与数据飞轮，无账号则无法沉淀 |
| **P1 应该有**（3 周内尽量做，可简化） | ③ 英文本地生活榜 | 解决"无英文大众点评"，但需 PGC 种子数据，可先做精简版（1–2 城、几十家精选） |
| **P1** | ④ 结构化攻略库 | SEO 获客主力；可与①清单共享内容底座，降低开发成本 |
| **P2 可以有**（MVP 期降级/延后） | ⑤ 旅行社区/问答 | 社区冷启动最难，且游客低频一次性——MVP 期用 **PGC 种子 + 轻量问答**替代全功能 UGC |

> ⚠️ **冲突暴露（需决策）**：「切入方向」把**社区**列为三大方向之一，但调研结论（4.3 节）明确"**不要一上来做全功能社区**、社区是长期飞轮、冷启动走 Reddit/YouTube 外部内容获客"。两者存在张力。当前处理：**社区方向保留但降级 P2，用 PGC 种子内容托底**，既守住长期价值，又不让 3 周预算被冷启动拖垮。若坚持社区 P0，需重新评估周期。

## 五、用户主流程（以场景 B「在地迷路型」为例）

一个外国游客从注册到解决一个实际问题、并完成价值沉淀的**完整闭环路径**：

```
① 发现      Reddit/YouTube 看到推荐 → 打开平台（英文 PWA / 轻 App）
   │
② 注册      邮箱 / Google 一键登录（无需中国手机号）───── 模块⑥
   │
③ 引导      选目的城市 + 行程天数 → 生成个性化「入境生存清单」── 模块①
   │
④ 预案      出发前勾选清单（装 VPN / 绑支付宝 / 装地图）→ 离线缓存
   │
⑤ 落地求助  街头打开 AI 助手 → 拍照翻译餐厅菜单 ──────────── 模块②
   │         提问「这家店能用 Visa 吗？怎么走最近？」
   │
⑥ 获得答案  AI 基于本地数据回答 + 关联「本地生活榜」该店信息 ── 模块②③
   │         （支付方式 / 评分 / 英文菜单标识）
   │
⑦ 决策记录  保存该店到收藏 → 写一条英文简评 ─────────────── 模块③⑥
   │         → 生成分享卡片发回 Reddit / 朋友
   │
⑧ 沉淀飞轮  评价与提问进入「本地榜 / 社区」────────────── 模块③⑤
            → 下次行程被推荐 → 形成「用得越多 → 数据越准 → 越离不开」飞轮
```

**这条流程串起了三大切入方向**：清单（准备）→ AI 助手（即时）→ 本地榜 + 社区（决策与沉淀）。

## 六、MVP 成功指标（North Star）

来源：调研 4.3 节，**避免用 DAU 误判低频场景**。

| 指标类型 | 指标 | 说明 |
|---|---|---|
| 🎯 **North Star** | AI 助手会话完成率 | 衡量核心差异化价值是否兑现 |
| 🎯 **North Star** | 用户主动分享率 | 衡量口碑获客与价值传递 |
| 辅助指标 | 清单完成率 | 衡量"入境生存"价值闭环 |
| 辅助指标 | 收藏 / 简评数 | 衡量数据飞轮启动信号 |
| 辅助指标 | 外部渠道回流转化率 | 衡量 Reddit/YouTube 冷启动效率 |

---

## 附：MVP 核心取舍小结

> **3 周只做 P0 三件套（清单 + AI 助手 + 账号），就能跑通"准备—解难—留存"完整闭环；P1 本地榜与攻略库补足"决策"，P2 社区作为长期飞轮待流量起来后再做重。** 不碰支付交易、不碰酒店机票（遵守约束），只做"信息 + 即时帮助"这一层。
